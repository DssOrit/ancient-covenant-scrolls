Load AI Provider Registry — read-only audit view

The shared registry that powers Load AI Core, AI Chat Studio, LoadStudio AI tools, X-VIDEO-AI, and X-AI-AUDIO. Every provider Load knows about is listed here, grouped by category, with its type, capabilities, and current status. Phase 1: read-only. Routing logic in load.js and image-prompt/index.html is unchanged.

Master rule: MVP uses free / open-source / local-first providers only. Paid providers are future-optional, off by default, never hard-coded, never exported.

Type legend: built-in ships with Load · local needs local server / GPU · free-api free public endpoint · cloud-optional user-supplied key, off by default · prompt-only emits structured prompt only · user-imported user provides asset · load-hosted-future Load-owned, not built yet.

Status enum: Not configured · Ready · Failed · Rate limited · Unsupported request · Returned no file · Offline · Local server unavailable · Needs user setup.
